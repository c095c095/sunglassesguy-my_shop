3.2.7 การออกแบบสถาปัตยกรรมระบบ (Architecture Design)

การพัฒนาระบบขายอุปกรณ์ตกปลาร้าน ก.กีฬา ออนไลน์ ถูกออกแบบให้อยู่บนพื้นฐานของระบบ Client/Server โดยพัฒนาในรูปแบบเว็บแอปพลิเคชัน (Web Application) ซึ่งเปิดโอกาสให้ผู้ใช้งานสามารถเข้าถึงระบบผ่านเว็บเบราว์เซอร์บนเครื่องคอมพิวเตอร์ สมาร์ตโฟน หรือแท็บเล็ตได้ โดยไม่ต้องติดตั้งโปรแกรมเพิ่มเติม ฝั่ง Client จะมีผู้ใช้เป็นทั้ง Admin (ผู้ดูแลระบบ) และ สมาชิก (ลูกค้า) โดยใช้ XAMPP Web Server เป็นตัวดำเนินการ และใช้ Server เป็นตัวประมวลผลและจัดการกับฐานข้อมูลต่างๆ ของระบบ

ระบบได้รับการออกแบบให้สอดคล้องกับความต้องการของผู้ใช้งาน เพื่อให้ระบบสามารถทำงานได้อย่างรวดเร็ว มีประสิทธิภาพ และรองรับการใช้งานในสภาพแวดล้อมของร้านค้าออนไลน์ได้อย่างเหมาะสม โดยแต่ละกลุ่มผู้ใช้จะมีบทบาทและหน้าที่ในการใช้งานระบบที่แตกต่างกัน เช่น การจัดการสินค้า การสั่งซื้อสินค้า การชำระเงิน และการจัดการคำสั่งซื้อ

สิ่งที่แสดงอยู่ในสถาปัตยกรรมของการพัฒนาระบบขายอุปกรณ์ตกปลาร้าน ก.กีฬา ออนไลน์ มีรายละเอียดดังต่อไปนี้

1. การออกแบบสถาปัตยกรรมเครือข่าย (Network Architecture)
   ระบบขายอุปกรณ์ตกปลาร้าน ก.กีฬา ออนไลน์ เป็นระบบที่พัฒนาขึ้นในรูปแบบเว็บแอปพลิเคชัน (Web Application) ซึ่งใช้สถาปัตยกรรมแบบ Client-Server โดยมีองค์ประกอบหลักดังต่อไปนี้
   - ฝั่งไคลเอนต์ (Client Side) ผู้ใช้งาน เช่น ผู้ดูแลระบบ (Admin) และสมาชิก (ลูกค้า) สามารถเข้าถึงระบบขายอุปกรณ์ตกปลาร้าน ก.กีฬา ออนไลน์ ได้ผ่านเว็บเบราว์เซอร์ (เช่น Chrome, Edge, Safari) บนเครื่องคอมพิวเตอร์ สมาร์ตโฟน หรือแท็บเล็ต โดยไม่ต้องติดตั้งโปรแกรมเพิ่มเติม ทำหน้าที่แสดงผลข้อมูล (User Interface) และรับคำสั่งจากผู้ใช้ เช่น การเลือกดูสินค้า การเพิ่มสินค้าลงตะกร้า การสั่งซื้อ และการแจ้งชำระเงิน โดยการส่งค่า Request ไปยัง Server
   - ฝั่งเซิร์ฟเวอร์ (Server Side) ทำหน้าที่เป็นศูนย์กลางในการประมวลผลข้อมูลและให้บริการแก่ไคลเอนต์ โดยเซิร์ฟเวอร์ทำงานด้วยซอฟต์แวร์ Apache HTTP Server ผ่าน XAMPP ใช้ภาษา PHP ในการประมวลผล เช่น การตรวจสอบสิทธิ์การเข้าใช้งาน (Login) การจัดการสินค้าและสต็อก การคำนวณราคาและค่าจัดส่ง และการจัดการสถานะคำสั่งซื้อ โดยการรับค่า Request จาก Client มาประมวลผล แล้วส่งค่า Response กลับไปยัง Client
   - ฐานข้อมูล (Database Server) ระบบใช้ฐานข้อมูลเชิงสัมพันธ์ (Relational Database Management System: RDBMS) โดยใช้ MySQL (MariaDB) เป็นระบบจัดการฐานข้อมูล และใช้ phpMyAdmin เป็นเครื่องมือในการบริหารจัดการฐานข้อมูล ฐานข้อมูลทำหน้าที่จัดเก็บข้อมูลหลักทั้งหมดของระบบ เช่น ข้อมูลผู้ใช้งาน ข้อมูลสินค้า ข้อมูลหมวดหมู่สินค้า ข้อมูลตะกร้าสินค้า ข้อมูลคำสั่งซื้อ ข้อมูลการชำระเงิน และข้อมูลแบนเนอร์ โดยมีการออกแบบความสัมพันธ์ของข้อมูลด้วย Primary Key และ Foreign Key เพื่อรองรับการทำงานและเชื่อมโยงข้อมูลระหว่างส่วนต่างๆ ของระบบอย่างถูกต้อง
   - การสื่อสารข้อมูล (Communication) การรับ-ส่งข้อมูลระหว่าง Client และ Server ใช้โปรโตคอล HTTP/HTTPS ผ่านระบบเครือข่ายอินเทอร์เน็ต

2. การออกแบบฐานข้อมูล (Database Design)
   การออกแบบฐานข้อมูลของระบบขายอุปกรณ์ตกปลาร้าน ก.กีฬา ออนไลน์ ได้ดำเนินการบนพื้นฐานของฐานข้อมูลเชิงสัมพันธ์ (Relational Database) โดยใช้ MySQL ในการพัฒนา ในการออกแบบฐานข้อมูล ได้คำนึงถึงการกำหนด Primary Key และ Foreign Key เพื่อสร้างความสัมพันธ์ระหว่างตาราง การทำ Normalization เพื่อลดความซ้ำซ้อนของข้อมูล และการกำหนดชนิดข้อมูลให้เหมาะสมกับลักษณะการใช้งานจริง โดยมีการจัดเก็บข้อมูลสำคัญ ได้แก่
   - ข้อมูลผู้ใช้งาน (user): รหัสสมาชิก, ชื่อผู้ใช้, รหัสผ่าน, อีเมล, ชื่อ-นามสกุล, เบอร์โทรศัพท์, ที่อยู่, ระดับสิทธิ์การเข้าถึง (permission)
   - ข้อมูลสินค้า (product): รหัสสินค้า, ชื่อสินค้า, รายละเอียด, รูปภาพ, ราคา, จำนวนสต็อก, รหัสประเภทสินค้า
   - ข้อมูลประเภทสินค้า (product_type): รหัสประเภท, ชื่อประเภท, รูปภาพ
   - ข้อมูลตะกร้าสินค้า (cart): รหัสตะกร้า, รหัสสมาชิก, รหัสสินค้า, จำนวน
   - ข้อมูลคำสั่งซื้อ (order): รหัสคำสั่งซื้อ, รหัสสมาชิก, วันที่สั่งซื้อ, ชื่อผู้รับ, เบอร์โทร, ที่อยู่จัดส่ง, ราคารวม, ค่าจัดส่ง, ประเภทการจัดส่ง, หมายเลขพัสดุ, สถานะ, หมายเหตุ
   - ข้อมูลรายละเอียดคำสั่งซื้อ (order_detail): รหัสรายละเอียด, รหัสคำสั่งซื้อ, รหัสสินค้า, ชื่อสินค้า, ราคาสินค้า, รูปสินค้า, จำนวน
   - ข้อมูลการชำระเงิน (payment): รหัสการชำระเงิน, รหัสคำสั่งซื้อ, รหัสธนาคาร, วันที่ชำระ, เวลาชำระ, รูปหลักฐาน, วันที่แจ้งชำระ
   - ข้อมูลธนาคาร (bank): รหัสธนาคาร, ประเภท, ชื่อบัญชี, เลขบัญชี
   - ข้อมูลแบนเนอร์ (banner): รหัสแบนเนอร์, รูปภาพ, สถานะ, วันที่อัปเดต
     นอกจากนี้ยังมีการกำหนดสิทธิ์การเข้าถึงข้อมูลตามบทบาทของผู้ใช้งาน เช่น ผู้ดูแลระบบ (Admin) และสมาชิก (ลูกค้า)

3. การออกแบบเอาต์พุต (Output Design)
   ระบบขายอุปกรณ์ตกปลาร้าน ก.กีฬา ออนไลน์ ได้ออกแบบการนำเสนอข้อมูลในรูปแบบเอาต์พุต เพื่อให้ผู้ใช้งานสามารถเข้าถึงและใช้ข้อมูลได้อย่างมีประสิทธิภาพ โดยเอาต์พุตของระบบประกอบด้วย
   - หน้าแรก (Homepage): แสดงแบนเนอร์โปรโมชั่น, หมวดหมู่สินค้า (เบ็ดตกปลา, คันเบ็ด, รอก, สายเอ็น, ทุ่น, เหยื่อปลอม, ตัวถ่วง, กล่องอุปกรณ์) และสินค้าแนะนำ
   - หน้ารายการสินค้า: แสดงสินค้าแยกตามหมวดหมู่ พร้อมรูปภาพ ชื่อ และราคา
   - หน้ารายละเอียดสินค้า: แสดงข้อมูลสินค้าอย่างละเอียด สถานะสต็อก และสินค้าที่เกี่ยวข้อง
   - หน้าตะกร้าสินค้า: แสดงรายการสินค้าในตะกร้า จำนวน ราคา และยอดรวม
   - หน้ายืนยันคำสั่งซื้อ: แสดงสรุปรายการสินค้า ที่อยู่จัดส่ง วิธีจัดส่ง ค่าจัดส่ง และยอดรวมสุทธิ
   - หน้าประวัติคำสั่งซื้อ: แสดงรายการคำสั่งซื้อทั้งหมด พร้อมสถานะ (รอชำระเงิน, รอตรวจสอบ, กำลังจัดส่ง, สำเร็จ, ยกเลิก)
   - Dashboard ผู้ดูแลระบบ: แสดงสรุปยอดคำสั่งซื้อ รายได้ สินค้าหมดสต็อก และรายการรอตรวจสอบ
   - รายงาน: รายงานสมาชิก, รายงานสินค้า, รายงานการขาย, รายงานการชำระเงิน, รายงานยอดรายได้, รายงานสินค้าขายดี, รายงานสต็อกสินค้า

4. การออกแบบอินพุต (Input Design)
   การออกแบบอินพุตของระบบขายอุปกรณ์ตกปลาร้าน ก.กีฬา ออนไลน์ ได้คำนึงถึงความสะดวกและความปลอดภัยในการใช้งานของผู้ใช้เป็นสำคัญ โดยมีการตรวจสอบความถูกต้องของข้อมูล (Validation) ทั้งในฝั่งผู้ใช้งานและฝั่งระบบ เพื่อช่วยลดข้อผิดพลาดที่อาจเกิดขึ้นจากการป้อนข้อมูล เช่น
   - แบบฟอร์มสมัครสมาชิก: กรอกข้อมูลชื่อ นามสกุล เบอร์โทร อีเมล ที่อยู่ ชื่อผู้ใช้ และรหัสผ่าน พร้อมยอมรับข้อกำหนดการใช้งาน
   - แบบฟอร์มเข้าสู่ระบบ: กรอกชื่อผู้ใช้และรหัสผ่าน
   - แบบฟอร์มจัดการสินค้า (Admin): เพิ่ม แก้ไข ลบข้อมูลสินค้า ประเภทสินค้า พร้อมอัปโหลดรูปภาพ
   - แบบฟอร์มจัดการสต็อก (Admin): ปรับปรุงจำนวนสต็อกสินค้า
   - ปุ่มเพิ่ม/ลด/ลบสินค้าในตะกร้า: ใช้ปุ่ม +/- และ X ในการจัดการจำนวนสินค้า
   - แบบฟอร์มยืนยันคำสั่งซื้อ: ข้อมูลที่อยู่จัดส่งถูกเติมอัตโนมัติจากโปรไฟล์ผู้ใช้ สามารถแก้ไขได้ พร้อมเลือกวิธีจัดส่ง
   - แบบฟอร์มแจ้งชำระเงิน: อัปโหลดรูปสลิปการโอนเงิน เลือกวันที่ เวลาที่ชำระ และเลือกธนาคารที่โอน
   - แบบฟอร์มจัดการคำสั่งซื้อ (Admin): อัปเดตสถานะคำสั่งซื้อ กรอกหมายเลขพัสดุ ยืนยัน/ปฏิเสธการชำระเงิน

5. ระบบของผู้ใช้งาน
   ระบบขายอุปกรณ์ตกปลาร้าน ก.กีฬา ออนไลน์ แบ่งระบบของผู้ใช้งานแยกเป็น 2 กลุ่ม คือ Admin (ผู้ดูแลระบบ) และ สมาชิก (ลูกค้า) ซึ่งแบ่งออกได้ดังนี้
   ฟอร์มของ Admin ใช้ในการดำเนินการเรื่องการเข้าใช้งาน มีดังนี้
   - เข้าสู่ระบบ มีไว้ตรวจสอบยืนยันตัวตนในการเข้าใช้ระบบ
   - บันทึก, ลบ, แก้ไขข้อมูลสินค้า, ข้อมูลประเภทสินค้า, ข้อมูลสต็อกสินค้า, ข้อมูลสมาชิก, ข้อมูลแบนเนอร์
   - จัดการคำสั่งซื้อ: อัปเดตสถานะ, ตรวจสอบการชำระเงิน, กรอกหมายเลขพัสดุ, ยกเลิกคำสั่งซื้อ
   - เรียกดูรายงานสมาชิก, รายงานสินค้า, รายงานการขาย, รายงานการชำระเงิน, รายงานยอดรายได้, รายงานสินค้าขายดี, รายงานสต็อกสินค้า
     ฟอร์มของ สมาชิก (ลูกค้า) ใช้ในการดำเนินการเรื่องการเข้าใช้งาน มีดังนี้
   - เข้าสู่ระบบ มีไว้ตรวจสอบยืนยันตัวตนของสมาชิก
   - สมัครสมาชิก สำหรับผู้ใช้ใหม่ที่ยังไม่มีบัญชีในระบบ
   - เรียกดูสินค้าและหมวดหมู่สินค้า
   - เพิ่มสินค้าลงตะกร้า จัดการจำนวนสินค้าในตะกร้า
   - ยืนยันคำสั่งซื้อและเลือกวิธีจัดส่ง
   - แจ้งชำระเงินพร้อมอัปโหลดหลักฐานการโอนเงิน
   - เรียกดูประวัติคำสั่งซื้อและติดตามสถานะ
   - แก้ไขข้อมูลส่วนตัวและเปลี่ยนรหัสผ่าน

6. เทคโนโลยีที่ใช้ในการพัฒนา
   PHP ใช้เป็นส่วนในการสร้าง Web Application ขึ้นมาเพื่อใช้งานในส่วนของ Admin และสมาชิก และยังใช้ในการเข้าถึงฐานข้อมูล MySQL อีกด้วย โดยใช้เทคโนโลยีต่างๆ ดังนี้
   - ภาษา PHP 8.2 สำหรับประมวลผลฝั่ง Server
   - HTML5, CSS3, JavaScript สำหรับส่วนติดต่อผู้ใช้ (User Interface) ฝั่ง Client
   - Bootstrap 5 สำหรับออกแบบ Responsive Design ให้รองรับหลายขนาดหน้าจอ
   - MySQL (MariaDB 10.4) สำหรับจัดเก็บข้อมูลทั้งหมดของระบบ
   - Swiper.js สำหรับแสดงแบนเนอร์และหมวดหมู่สินค้าแบบ Carousel

   ใช้ XAMPP Web Server ในการทำงานบนฝั่ง Server ทำหน้าที่ในการรับคำสั่งจากการร้องขอของฝั่ง Client (โดยผ่านทาง Browser) และประมวลผลการทำงานจากการร้องขอดังกล่าว แล้วส่งข้อมูลกลับไปยังเครื่องของ Client ที่ร้องขอ โดยระบบใช้ Apache HTTP Server เป็นเว็บเซิร์ฟเวอร์หลัก และใช้ phpMyAdmin เป็นเครื่องมือในการบริหารจัดการฐานข้อมูล

8. สรุปแนวทางการออกแบบสถาปัตยกรรม
   การออกแบบสถาปัตยกรรมของระบบขายอุปกรณ์ตกปลาร้าน ก.กีฬา ออนไลน์ ได้ดำเนินการโดยมุ่งเน้นให้ระบบสามารถทำงานได้อย่างมีประสิทธิภาพ มีความถูกต้องของข้อมูล และรองรับการใช้งานของผู้ใช้หลายระดับ ระบบถูกพัฒนาในรูปแบบเว็บแอปพลิเคชันโดยใช้สถาปัตยกรรมแบบ Client-Server เพื่อให้ผู้ใช้งานสามารถเข้าถึงระบบผ่านเว็บเบราว์เซอร์ได้โดยไม่จำเป็นต้องติดตั้งโปรแกรมเพิ่มเติม อีกทั้งยังมีการออกแบบฐานข้อมูล การรับข้อมูลเข้า และการแสดงผลข้อมูลอย่างเป็นระบบ เพื่อสนับสนุนการบริหารจัดการร้านค้าออนไลน์ให้มีความสะดวก รวดเร็ว และเหมาะสมกับการใช้งานจริง ระบบใช้สถาปัตยกรรมแบบ Web Application ที่เน้นการบริหารจัดการข้อมูล (Information System) โดยไม่มีความซับซ้อนด้านฮาร์ดแวร์ภายนอก ข้อดีคือสามารถติดตั้งและใช้งานได้ทันทีบนอุปกรณ์ที่มีอยู่เดิม ดูแลรักษาง่าย (Maintenance) ดังแสดงในภาพที่ 3.x
